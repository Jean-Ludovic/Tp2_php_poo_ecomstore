DIAGRAMME DE CAS D'UTILISATION

Analyse du Diagramme de Cas d'Utilisation

Ce diagramme de cas d'utilisation représente les interactions entre les deux principaux acteurs d'une application de commerce en ligne : l'utilisateur et l'administrateur. Il met en évidence les différentes fonctionnalités accessibles à chacun de ces acteurs.

1. Acteur : Utilisateur
L'utilisateur de l'application a accès aux cas d'utilisation suivants :
- Browse Products : L'utilisateur peut parcourir les produits disponibles sur la plateforme.
- Add to Cart : L'utilisateur peut ajouter des produits à son panier pour les acheter plus tard.
- Make Purchase : L'utilisateur peut finaliser l'achat des produits présents dans son panier.
- Register : L'utilisateur peut s'inscrire sur la plateforme pour créer un compte.
- Login : L'utilisateur peut se connecter à son compte pour accéder aux fonctionnalités de l'application.

2. Acteur : Administrateur
L'administrateur de l'application a accès aux cas d'utilisation suivants :
- Manage Products : L'administrateur peut gérer les produits disponibles sur la plateforme, y compris l'ajout, la modification, et la suppression de produits.
- Manage Users : L'administrateur peut gérer les comptes des utilisateurs, incluant la création, la modification, et la suppression de comptes.
- View Payments : L'administrateur peut consulter les paiements effectués par les utilisateurs pour surveiller et gérer les transactions financières.

Ce diagramme met en lumière les interactions principales entre les acteurs de l'application et les fonctionnalités qu'ils peuvent utiliser, offrant ainsi une vue d'ensemble des responsabilités et des actions possibles dans le système.
